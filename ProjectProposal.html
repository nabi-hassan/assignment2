  1. Overview

In a world of virtual media where the internet has become the foremost invention, which is used to connect the entire world. Chat rooms are a vital communication application, used in both social and work-based interactions. Since it is real time, people can get notified about messages instantly. This project is an extension of existing chat applications, which will be more user friendly. Various projects like e-commerce, social media etc. all depend on chat applications.

  1. Project Detail

  1. Project Goals

  • Functional goals: This application will allow for chatting amongst user via the internet. It will also allow users to create or join chat rooms and talk accordingly. It will allow successful delivery of messages across users and rooms.
  • Technological goals: Python will be the main medium for implementing our system. Due to its numerous libraries and versatility, it can aid us in implementing this system. It will run on a local network which can be further deployed into Realtime.
  • Quality goals: UI will be user friendly and easy to grasp for even novice people and will be fast and efficient with no lags in sending or receiving messages.
  • Organizational goals: For real time deployment we aim to try Heroku as a new technology to deploy python-based applications. However, before we shall apply local testing to see if all functionality is up to mark and then try to deploy and test on real time accordingly
  • Constraints: The constraints in this project are regarding security and encryption of messages which is a completely different field of experience, however due to the anonymous nature of the application, its not needed as much.

  1. Project Scope

Our project will revolve around people hosting one or many chatrooms where people can join and choose whether to be anonymous or not. Each chat room can have a purpose(title) which will attract people of similar people to join and participate. The anonymity will help people express themselves and talk freely. However, this project can have modifications if needed

  1. Existing Work

Currently there are various chat systems in place like Facebook and whatsapp, however, these systems require verification and your identity is known, however, we aim to create a space where the user can sign up and login with an account, but their name and information remains anonymous.

  1. User Stories

In terms of user stories, the user shall first sign up/login in with an email or id of sorts. After he/she is logged in, he can see various people who are online or joined the chatroom. The user shall can either choose to make a new chatroom which others can join, or join any of the existing chats room and chat to his or her heart’s content.
